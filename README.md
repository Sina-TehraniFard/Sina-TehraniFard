## 概要
バックエンドを主軸に、要件定義〜結合テストまで一通り経験。災害時支援システム、証券システム、BtoB SaaS などの開発に携わりました。  個人開発では、OpenAI API を用いた「AIタロット占い」を公開し、次は RAG を設計・実装予定です。

## 開発スタイル
現在は、LLMを前提とした開発プロセスで開発しています。LLMの性能向上により、実装よりも設計や判断、レビューに人の時間を使うほうが合理的だと考えています。

そのため、自分でコードを書くことをあえて制限し、実装は基本的にLLMに任せています。空いた時間は、言語仕様の理解や設計の勉強に充てています。

実装には Claude Code を使用し、出力のばらつきを抑えるためにカスタムコマンドを定義しています。トークン消費は増えますが、タスクを細かく分割し、会話を短く区切ることで対応しています。

カスタムコマンドは「思考」「調査」「設計」「実装」「レビュー」の5フェーズに分けています。1サイクルごとに会話を切り替え、コメントや設計メモは継続的に更新しています。

現時点では、エージェントによる完全自動化は行っていません。人間とLLMを組み合わせた形のほうが、現時点では爆速開発できると確信があるからです。
